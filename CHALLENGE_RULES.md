# Proof Before Consequence™ Challenge — Rules

This is intentionally simple:

You’re competing against **autopilot**.

Autopilot doesn’t wait for meetings.

---

## The core challenge

For each level:

1) **Decision lock** (Step 1)
   - pick the right outcome: `ALLOW`, `ALLOW WITH BOUNDS`, `ESCALATE`, `BLOCK`, `SAFE DEGRADE`

2) **Authority + bounds** (Step 2)
   - set the correct blast‑radius controls
   - route to the correct accountable humans

3) **Proof lock** (Step 3)
   - two proof packs are presented
   - one has been altered
   - only the authentic one verifies

If you verify the authentic pack, you get a **flag**.

---

## Win conditions

- You finish within the time limit.
- You verify the authentic pack.
- You capture the flag.

---

## Expert mode (optional)

> Try to change the proof pack *and still pass verification*.

This is the point of the product.

If you find a real weakness, report it responsibly to **founder@judgementspine.com**.

---

## Notes

- All scenarios are synthetic.
- The proof packs are intentionally small so verification runs in a browser.
- Verification is based on a signed manifest + per‑file SHA‑256 hashes.
